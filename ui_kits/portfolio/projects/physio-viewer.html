Context

Clinical data is useless without visualization.

The PULSAR wearable captures continuous ECG and PPG data from patients during clinical trials. But raw binary data stored on an SD card or uploaded to AWS has no value until a clinician can see it, validate it, and export it.

PHYSIO VIEWER is a Qt/QML desktop application built to give clinicians and engineers a real-time visualization interface for PULSAR data. It connects directly to the device via USB for live extraction, or pulls stored sessions from AWS for retrospective analysis — and exports everything to CSV.

Features

Real-time curves. Dual source. Full export.

Real-time ECG / PPG
Live scrolling curves of ECG and PPG signals from the PULSAR device — rendered in QML with hardware acceleration. Configurable time window and amplitude scale.
USB extraction
Direct connection to the PULSAR device via USB. Streams live data or extracts recorded sessions from the onboard SD card in seconds.
AWS cloud sync
Pull any recorded session from AWS S3 for retrospective visualization. Authenticated access with session browser and timeline navigation.
CSV export
Export any session or time range as CSV — compatible with MATLAB, Python, and standard clinical analysis tools.
Multi-signal display
Simultaneous visualization of multiple physiological signals — ECG, PPG (multi-wavelength), accelerometer — in synchronized panels.
Clinical annotations
Add timestamped annotations directly on the signal timeline — events, observations, quality flags — saved with the session data.

Architecture

Qt/QML + C++ backend. Two data sources.

The application follows a strict Qt MVC pattern — a C++ backend handles data acquisition (USB serial or AWS S3 SDK), signal processing, and file I/O, while a QML front-end renders the live curves and UI with hardware acceleration. The two layers communicate via Qt signals and properties.
