Context

The firmware is only half the product.

An embedded system without a usable software layer is a prototype no one will adopt. The BABYCAM hardware streams H.264 video over WiFi — but parents need an app that receives it, surfaces alerts, and makes sense of the sensor data, all in real time.

Built as the companion to BABYCAM, this Flutter application handles stream decoding, AI alert delivery, environmental monitoring display, remote device control, and cloud sync — targeting both iOS and Android from a single Dart codebase.

Features

Six features. One screen.

Live stream
1080p@30FPS H.264 stream decoded and rendered in real time — under 200ms end-to-end latency from sensor to screen.
AI cry detection
On-chip AI events pushed instantly to the app — cry, movement, silence. Push notifications via AWS SNS when the app is in background.
Environmental monitoring
Live temperature, humidity, air quality, and luminosity readings from the 4 embedded sensors, updated in real time alongside the video feed.
Remote controls
Remotely trigger the integrated nightlight and play lullabies through the Class D amplifier — without waking the baby.
AWS cloud sync
Environmental data and event logs synced to AWS for long-term storage, multi-device access, and parental reporting.
History & stats
Sleep pattern history, cry frequency charts, and room condition trends — giving parents actionable data over time.
Flutter Dart iOS Android H.264 decode WiFi streaming AWS Push notifications Real-time UI
Architecture

Flutter + WiFi + AWS. One codebase, two platforms.

The app communicates directly with the BABYCAM device over the local WiFi network for the video stream and control commands. Cloud features (storage, notifications, history) are handled via AWS — S3 for data, SNS for push alerts, and a lightweight Lambda backend.
