Context

Consumer sleep trackers show you a score. We built the signal.

Most wearables abstract away the underlying physiology — you get a colored ring or a "sleep score" with no transparency. TheSleepGuard was built from the sensor layer up: raw PPG signal, digital filtering, HRV computation, and real-time sleep phase classification, all running on a microcontroller worn on the wrist.

Built during a 6-month international exchange between ESME SUDRIA (Paris) and UDESC (Joinville, Brazil), August–December 2023. Academically validated for M1 Embedded Systems. Two engineers, one supervisor, one wristband.

Hardware

MAX30105. OLED 128×64. SD card.

The core sensor is the MAX30105 multi-spectral PPG — it captures the light absorption signature of blood flow through the wrist, providing both heart rate and the beat-to-beat variability (HRV) that sleep phase classification depends on. An infrared threshold detects whether the sensor is worn correctly before recording.

Signal processing

MATLAB filter design. Arduino implementation.

The raw PPG signal is noisy — motion artifacts, ambient light, hardware noise. The processing pipeline starts with FFT analysis to identify the dominant frequencies, then applies a FIR low-pass filter (order 30, cutoff 0.1 Hz) designed in MATLAB and implemented as convolution coefficients in the Arduino firmware.

Sleep phases

Four phases. One HRV threshold per transition.

Sleep phase classification is based on heart rate thresholds derived from HRV analysis. Each phase has a distinct BPM range — the firmware classifies in real time and displays both the current phase and cumulative duration per phase on the OLED.
