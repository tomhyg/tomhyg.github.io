Read the README.md file within this skill, and explore the other available files (colors_and_type.css, ui_kits/portfolio/, preview/, assets/).

If creating visual artifacts (slides, mocks, throwaway prototypes, etc), copy assets out and create static HTML files for the user to view. Always link `colors_and_type.css` and use the CSS variables from it — never hardcode hex.

If working on production code, read the rules in README.md (Voice, Casing, Visual Foundations, Iconography) and become an expert in designing with this brand. Pay particular attention to:

- Bilingual FR / EN copy with mirrored content (no slash-mixing)
- Dark-first surface system, violet electric accent used sparingly
- Geist + Geist Mono typography
- No emoji, no gradient sweeps, no "rocket / wizard" copywriting
- 1px hairline borders, restrained radii, no thick frames
- Lucide icons at 1.5px stroke

If the user invokes this skill without any other guidance, ask them what they want to build or design, ask some questions, and act as an expert designer who outputs HTML artifacts _or_ production code, depending on the need.
